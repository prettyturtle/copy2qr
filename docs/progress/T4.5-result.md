## 구현 결과: T4.5 HomePage 통합

### 상태: OK

### 변경 파일
- `src/pages/HomePage.tsx` (수정): 빈 스켈레톤 → 완전한 메인뷰 통합 구현.

  **상태 관리:**
  - `pasteData: PasteData | null` — 현재 붙여넣기 데이터
  - `shareUrl: string` — 생성된 공유 URL
  - `shareDisabled: boolean` — "미리보기만" 선택 시 QR/공유 비활성화
  - `pendingImageData: PasteData | null` — 크기 초과 이미지의 사용자 선택 대기 상태

  **통합 플로우:**
  1. `PasteZone` → `handlePaste`: 이미지이면 `isImageOversized()` 체크. 초과 시 `pendingImageData` 설정 → `ImageSizeDialog` 표시. 정상이면 `pasteData` + `shareUrl` 즉시 설정.
  2. `ImageSizeDialog`: "줄여서 공유" → `resizeImage(300, 300, 0.7)` 후 `pasteData`/`shareUrl` 갱신. "미리보기만" → `shareDisabled=true`, `shareUrl=""`.
  3. `Preview` → `handleContentChange`: `pasteData.content` 업데이트 + `shareDisabled` 아닐 때 `shareUrl` 재생성.
  4. `QRCodeDisplay` (`ref=qrRef`): `shareUrl` 전달.
  5. `ActionBar`: `onCopyLink` → `copyShareLink(shareUrl)` + `showToast`. `onDownloadQR` → `qrRef.current.downloadQR()`. `onReset` → 전체 상태 초기화.
  6. `ToastContainer`: `toasts`/`removeToast` 연결.

  **ImageSizeDialog**: 별도 Modal 없이 인라인 글래스 카드. amber-400 경고 아이콘 + 설명 텍스트 + 2개 버튼(줄여서 공유/미리보기만). `role="alertdialog"` + `aria-labelledby`/`aria-describedby` 접근성 적용.
