### 구현 세부사항

#### useClipboard.ts — isSupported 반환
- `typeof ClipboardEvent !== "undefined"`로 지원 여부 판별
- 미지원 시 `useEffect`에서 paste 리스너 등록 스킵
- 기존 핸들러 로직 무변경

#### useShareData.ts — execCommand 폴백
- `navigator.clipboard && typeof navigator.clipboard.writeText === "function"` 체크
- 미지원 시: textarea 생성 → body append → focus/select → execCommand("copy") → body remove
- execCommand 실패 시 throw (호출처 catch로 처리)

#### HomePage.tsx — 두 가지 에러 처리 추가
1. 클립보드 미지원 배너: `typeof ClipboardEvent !== "undefined"` 판별 (PasteZone과 이중 등록 없이 독립 감지), `role="alert"` 접근성 준수
2. 리사이즈 후 URL 초과: `handleResize` 내부에서 `isImageOversized(resized, IMAGE_MAX_BYTES)` 재검사 → 초과 시 `shareDisabled=true`, toast 안내, 미리보기 전환

#### QRCodeDisplay.tsx — QR 생성 실패 처리
- `useState<boolean>(false)` qrError 상태 추가
- `useEffect([value])`: value 변경 시 qrError 초기화
- `QRCodeCanvas`의 `onError` prop으로 실패 감지 → qrError=true
- qrError=true 시: 에러 안내 div(`role="alert"`) 렌더링, downloadQR 비활성화
